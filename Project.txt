Project 기획 문서
    해당 프로젝트 목적 및 설명 
        이 프로젝트는 1인 자취생, 요리초보 등 요리를 하고 싶지만 요리를 모르는 사람들을 위해서 만드는 프로젝트 입니다.
        챗봇을 통하여 사용자와 요리를 대화하며 식재료, 원하는 음식, 요리순서등 알려주는 시스템이다.
    
    사용 언어
        python 

    흐름도 

        데이터 수집
            ↓
        데이터 가공 
            ↓
        데이터 저장 
            ↓
        챗봇과 대화(만약 데이터 저장된 데이터에 사용자가 원하는 데이터 존재x면 데이터 수집으로 돌아감)

    각 파일 설명 
        Crawling.py
            Crawling.py 모듈은 웹스크래핑을 통해서 레시피의 재료를
            추출하는 것을 목표로하는 웹 스크래핑 모듈이다.  
            카테고리 리스트를 순회하며, 각 키워드당 N개의 레시피를 비축

        RecipeChatBot.py
            DB에서 데이터를 로드하고, 사용자가 누구인지(페르소나)에 따라 지능적으로 대응

        CollectData.py
            봇을 켜기 전, 혹은 정기적으로 데이터를 업데이트할 때 실행하는 공급망
        
        driver_loader.py
            Selenium WebDriver 설정을 모듈화하여
            크롤링 코드에서 재사용
        
        data_frame_factory.py
            이 모듈은 수집된 dict 형태의 리스트를 받아서 정형화된 DataFrame으로 변환하고 저장하는 역할을 합니다.

    확장성 
        향후 칼로리, 영양성분, 재료의 가격등을 추가할 예정임
        가격 및 영양성분을 matplotlib를 사용하여 도넛 차트 및 그래프를 활용하여 시각화 예정
