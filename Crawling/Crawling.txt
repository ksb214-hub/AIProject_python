Crawling.py 설계문서 

1. 개요 (Overview)
    Crawling.py 모듈은 웹스크래핑을 통해서 레시피의 재료를
    추출하는 것을 목표로하는 웹 스크래핑 모듈이다. 
    사용자가 입력한 검색어를 기반으로 레시피의 제목, url를 추출하여 
    향후 상세 페이지로 이동하여 재료 및 레시피 데이터를 추출할 예정이다.


2. 역할 
    Crawling.py
        웹 페이지에서 레시피 제목과 url를 추출한다. 
        레시피 제목과 url를 바탕으로 향후 해당 레시피와 재료를 추출한다.

3. 사용기술
    사용 라이브러리 
        Python 
            Selenium
            bs4
            webdriver_manager
            time


4. 순서도 
    1. 사용자에게 input을 통해 검색어를 입력 받는다.
    2. driver를 셋팅한다.
    3. 검색 URL 뒤에 검색어를 붙여 검색 결과를 가져온다
    4. 레시피 개수 selector를 통해
        4.1 레시피 개수가 0개이면 종료
    5. 레시피 개수가 1개 이상일 경우 검색 결과 페이지에 표시된 모든 레시피 카드 요소를 가져온다.
    6. 데이터 추출(레시피 제목, 레시피 URL)
    -> 이 데이터는 각각 리스트에 저장된다
        titles.append(title)
        urls.append(url)

5. 사용 URL 
    검색 페이지
        https://www.10000recipe.com/recipe/list.html?q=
        검색어를 URL 뒤에 붙여 레시피 검색 결과를 가져온다.

        

    예시
        https://www.10000recipe.com/recipe/list.html?q=김치


    상세 페이지
        https://www.10000recipe.com
        상세 레시피 페이지는 상대 경로 형태로 제공되기 때문에 base URL과 결합하여 사용해야 한다.    


6. 주요 Selector 정보
    Crawling.py는 다음 selector를 사용하여 데이터를 수집한다.

    레시피 개수 selector        
        #contents_area_full > ul > div > b
        검색 결과의 레시피 개수를 확인하는 selector이다.



        주의사항
            해당 selector는 DOM 구조에 의존하기 때문에 사이트 구조가 변경될 경우 동작하지 않을 수 있다.
            향후 class 기반 selector로 개선이 필요하다.



    레시피 목록 selector
        #contents_area_full > ul > ul > li
        검색 결과 페이지에 표시된 모든 레시피 카드 요소를 가져온다.



    재료 selector
        #divConfirmedMaterialArea > ul:nth-child(1) > li
        레시피 상세 페이지에서 재료 목록을 추출하는 selector이다.


시니어의 조언 (Technical Review)
    Selector의 유연성
        작성해주신 #contents_area_full > ul > ul > li는 구조가 조금만 바뀌어도 깨지기 쉽습니다.
        만개한레시피의 경우 레시피 카드가 공통적으로 common_sp_list_li 클래스를 가집니다. 
        추후 문제가 생기면 soup.select('.common_sp_list_li') 형태로 변경하는 것을 추천합니다.

    Anti-Bot 회피
        백그라운드(headless) 실행 시 서버에서 봇으로 감지하고 차단할 수 있습니다. 
        위 코드에 user-agent 설정을 넣어두었으니 차단을 최소화할 수 있을 겁니다.

    데이터 가공(re) 준비
        추후 상세 페이지에서 재료를 긁어올 때 100g, 2큰술, 약간 같은 텍스트가 섞여 들어올 텐데, 
        이때 정규표현식(re)을 사용하여 **[재료명, 수량, 단위]**로 파싱하는 로직을 별도 모듈로 분리하면 챗봇 답변 생성 시 훨씬 깔끔한 데이터를 줄 수 있습니다.

    해결 방안 
        1. 패키지 초기화 (__init__.py) 추가
            파이썬은 특정 폴더를 "패키지"로 인식하려면 내부에 __init__.py 파일이 있어야 함 
            (Python 3.3+ 이후로는 필수는 아니지만, 명시적 구분과 하위 호환성을 위해 시니어들은 습관적으로 만듬)
            driver/ 폴더 안에 비어있는 파일인 __init__.py를 생성
            utils/ (혹은 다른 폴더)가 있다면 거기에도 생성

        2. 실행 경로(CWD) 이슈 해결 (가장 추천하는 방법)
            터미널에서 AIProject 위치에 있는데 python Crawling/Crawling.py를 실행하면,
            파이썬은 Crawling 폴더 내부만 먼저 뒤짐 
            driver 폴더가 한 단계 위에 있다면 찾지 못하는 게 당연함
            이럴 땐 -m (module mode) 플래그를 사용하여 프로젝트 루트에서 실행
            
            터미널
