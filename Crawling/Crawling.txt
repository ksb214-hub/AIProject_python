Crawling.py 문서화 

전체 흐름
    검색 → 리스트 수집 → 선택 → 상세 → 재료 추출 → 정제 → DataFrame → 추천 시스템

목표
    웹스크래핑을 통하여 레시피 데이터를 수집하고,
    수집된 데이터를 기반으로 추천 시스템에 활용하는 것을 목표로 한다.

사용 언어
    Python


사용 라이브러리
    Selenium
	BeautifulSoup
	Pandas
	time
	re


크롤링 대상 URL
    첫 번째 파싱 URL (검색 페이지)
        https://www.10000recipe.com/recipe/list.html?q=
    
    두 번째 파싱 URL (상세 페이지)
        https://www.10000recipe.com
    
    Selector 정보
        레시피 개수 selector
            #contents_area_full > ul > div > b
            해당 구조는 사이트 변경 시 깨질 수 있음 → class 기반 selector로 개선 필요
        
        URL selector
            #contents_area_full > ul > ul > li
        
        재료 selector
            #divConfirmedMaterialArea > ul:nth-child(1) > li

순서도
1. 검색어 입력
    검색어 변수: search

2. driver 설정
    백그라운드 실행 (headless 모드)

3. 검색 페이지 접속
    driver.get(첫 번째 파싱 url + search)

4. 레시피 개수 파싱
    selector를 이용하여 레시피 개수 추출

5. 레시피 존재 여부 판단
    5-1. 레시피 개수가 1개 이상일 경우
	    for 문을 통해 리스트 순회

        5-1-1. URL 파싱
            selector: div > a
            href 추출
        
        5-1-2. 제목 파싱
            selector: div.common_sp_caption > div.common_sp_caption_tit.line2
        
        5-1-3. 데이터 저장
            title, href 리스트에 append
        
    5-2. 레시피 개수가 0일 경우
        해당 레시피가 존재하지 않습니다.
        break

6. 레시피 목록 출력
    [1] 김치찌개  
    [2] 된장찌개  
    [0] 종료  
    [auto] 자동 수집  


7. 사용자 입력 처리

    7-1. 0 입력 시
        프로그램 종료 (driver 종료)
        break

    7-2. auto 입력 시
        상위 N개 레시피 자동 수집

    7-3. 숫자 입력 시
        해당 index 레시피 선택
        index 보정 필요 (idx - 1)


8. 상세 페이지 이동
    if href.startswith("http"):
        detail_url = href
    else:
        detail_url = detail_base_url + href
    
    상대경로 / 절대경로 처리 필수

9. HTML 파싱
    BeautifulSoup 사용

10. 재료 추출
    selector 기반 반복문 수행

    10-1. 재료 정제 로직
        text = ing.get_text(strip=True)
        text = text.replace("구매", "")
        text = re.split(r"\d", text)[0]

    	‘구매’ 제거
        용량(숫자) 제거
        재료명만 추출

11. 재료 리스트 저장
    list 형태로 append

12. 데이터 누적
	선택된 레시피의 재료를 누적 저장
	자동 수집 시에도 동일하게 누적

13. DataFrame 변환
    pandas DataFrame 생성

14. 결과 출력
    재료 데이터 출력

15. 추천 시스템 실행
	- 사용자 보유 재료 입력
	- 레시피 재료와 비교

    추천 방식
	    교집합 기반 유사도 계산
        유사도 = (겹치는 재료 수) / (전체 재료 수)
        유사도 높은 순으로 정렬 후 출력


흐름 요약
    [입력]
    검색어 입력

        ↓
    [크롤링]
    레시피 목록 수집

        ↓
    [선택]
    - 개별 선택
    - 자동 수집
    - 종료

        ↓
    [데이터 처리]
    재료 추출 + 정제 + 누적

        ↓
    [데이터 구조화]
    DataFrame 생성

        ↓
    [추천 시스템]
    사용자 재료 입력 → 유사도 계산 → 추천 출력


핵심 변경 포인트
    driver 종료 위치 수정
	    - 기존: 중간 종료 → 오류 발생 가능
	    - 수정: 마지막에 종료 (finally)

    두 번째 URL 처리 방식
        if href.startswith("http"):
            detail_url = href
        else:
            detail_url = detail_base_url + href

        - 사이트 구조에 따라 상대/절대 경로 처리 필수
    
    최신 Selenium 반영
        driver = webdriver.Chrome()

        - webdriver-manager 없이 자동 실행 가능

    li 기준 반복 구조
        driver.find_elements(By.CSS_SELECTOR, "#contents_area_full > ul > ul > li")
        
        - 모든 레시피 카드 수집

    내부 a 태그 접근
        item.find_element(By.CSS_SELECTOR, "div > a")
        
        - 구조 변경에도 비교적 안정적
    
    URL 리스트화
        urls.append(href)
    
    백그라운드 실행
        options.add_argument("--headless")
        - 속도 향상
	    - 리소스 절약
	    - 서버 환경에 적합
    
    재료 추출 핵심 로직
        text = ing.get_text(strip=True)
        text = text.replace("구매", "")
        text = re.split(r"\d", text)[0]
    
    종료 기능
        if idx == 0:
            driver.quit()
            return
    
    사용자 입력 UX 개선
        [1] 김치찌개
        [2] 된장찌개
        [0] 종료
        [auto] 자동 수집
    
    index 보정
        selected_url = urls[idx - 1]
